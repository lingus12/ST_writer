# ST_writier
세특 작성에 속도와 정확성과 학생의 희망을

성취평가 결과분석 및 세특 작성기

교사용 오프라인 단일 HTML 도구입니다. 수행·정기시험 성적과 성취수준을 한 곳에서 정리하고, 평가 근거에 기반한 세특(교과 세부능력 및 특기사항) 문장 작성까지 이어집니다.

주요 기능

작성 준비 — 학번 명렬(성명 미사용), 평가 비율표 자동 인식(새 양식의 성취기준·평가요소까지)
수행평가 — 계획(채점 요소·기준점수·성취수준별 성취수준), 나이스 강의실별 일람표 xlsx 그대로 업로드, 과제별 코멘트
정기시험 — 문항정보표·교과목별 학생답 정오표 xlsx 업로드, 서답형 채점기준표(교육부 형식), 정답률·변별도·신뢰도 분석과 성취도 분포(막대 클릭 시 수준별 학생 상세)
세특 입력기 — ① 교사 종합평가(관찰 기록·교사 A~E 평가) → ② 생성·조망(수준별 분량, 금기어 순화, xlsx 왕복) → ③ 외부 AI 프롬프트(복사·붙여넣기 방식, 전면 비식별)


개인정보 · 오프라인 원칙 (v1.1.0 기준)


외부 전송 없음 — 이 도구에는 외부 서버로 데이터를 보내는 코드가 없습니다. 과거 버전의 AI 직접 연동 기능(OpenAI·Gemini·로컬 LLM 설정 UI 포함)은 코드 자체가 제거되었습니다.
브라우저 저장 없음 — 작업 내용은 브라우저(localStorage 등)에 저장되지 않습니다. 창을 닫으면 화면의 데이터는 사라지며, 사용자가 직접 저장한 .json 파일에만 존재합니다. 공용 PC에서도 데이터가 남지 않습니다. (실행 시 구버전이 남긴 브라우저 저장 데이터가 있으면 자동 삭제합니다.)
성명 미저장 — 학생 성명은 저장되지 않으며 학번(또는 반/번호)만 사용합니다.
외부 AI는 복사·붙여넣기 방식 — 자동 연동이 아니라 교사가 프롬프트를 직접 복사해 붙여넣는 방식입니다. 프롬프트 생성 시 명렬의 모든 학번이 프롬프트 전체(교사 메모·문체 예시·활동지 포함)에서 임시번호(S01…)로 자동 치환되고, 결과를 붙여넣으면 복원됩니다. 단, 실명·별명 등 문자 정보는 자동 탐지되지 않으므로 복사 후 내용을 한 번 확인하고, 외부 AI 사용 전 소속 기관의 개인정보 보호 지침을 따르세요.
⚠ 이 저장소에는 학생 데이터 파일(JSON·xlsx)을 절대 올리지 마세요. (.gitignore로 차단되어 있습니다)

사용 흐름 (요약)

도구를 열면 홈 화면에서 입력 범위(수행/지필)를 고르고 시작할 데이터를 선택합니다. 기존 작업은 [💾 저장 파일 불러오기]로 이어집니다.
학번 명렬 → 나이스 xlsx 업로드(수행성적·문항정보표·정오표) → 세특 생성 → 검토·수정 → xlsx 내보내기.
자동저장이 없으므로 작업 중과 종료 전에 반드시 💾 저장(.json)을 눌러 파일을 보관하세요. (Ctrl+S 지원, 저장하지 않고 닫으면 경고가 표시됩니다)
생성 결과는 초안입니다. NEIS 입력 전 교사가 사실관계와 문장을 반드시 검토·수정하세요.
